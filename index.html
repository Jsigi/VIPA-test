SQL-Definition

Beispielhafte Tabellenstruktur

So könnten deine Tabellen in PostgreSQL oder SQLite aussehen.

CREATE TABLE owners (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,
  email TEXT NOT NULL
);

CREATE TABLE dogs (
  id INTEGER PRIMARY KEY,
  owner_id INTEGER REFERENCES owners(id),
  name TEXT NOT NULL,
  breed TEXT,
  vaccinated BOOLEAN DEFAULT false,
  next_vaccine_due DATE
);

CREATE TABLE courses (
  id INTEGER PRIMARY KEY,
  title TEXT NOT NULL,
  next_session DATE
);

CREATE TABLE enrollments (
  dog_id INTEGER REFERENCES dogs(id),
  course_id INTEGER REFERENCES courses(id),
  enrolled_at DATE DEFAULT CURRENT_DATE
);

CREATE TABLE documents (
  id INTEGER PRIMARY KEY,
  dog_id INTEGER REFERENCES dogs(id),
  file_name TEXT,
  file_type TEXT,
  uploaded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Technologien

Stack-Empfehlung

Alles, was du für die Umsetzung benötigst.

  • Frontend: HTML, CSS, JavaScript (hier Vanilla; React/Vue optional)
  • Backend: Node.js oder Python (Express/FastAPI) für REST- oder GraphQL-APIs
  • Datenbank: PostgreSQL oder SQLite für relationale Daten & Foreign Keys
  • Storage: S3/Blob Storage oder Datenbank-Files für Dokument-Uploads
  • Auth: JWT-basierte Session oder OAuth2, falls Zugänge nötig sind
  • Infra: Docker-Container für lokale Entwicklung & Deployment
